Comandos do GitHub

git init

Inicialização do repositório na pasta atual.
Cria uma pasta oculta .git onde tem as configurações do repositório criado.

git status

Exibe estado atual do repositório Git.
Mostra informações importantes sobre o estado do repositório Git, como o nome do branch atual, os arquivos que foram modificados desde o último commit, os arquivos que foram adicionados ou excluídos, e se existem arquivos que foram modificados mas ainda não foram adicionados para um próximo commit.

git add name.file || git add .

Adiciona arquivos para o fluxo de versionamento.
Quando informado o nome do arquivo ele adiciona apenas o arquivo informado, se tiver vários arquivos a serem enviados para o fluxo de versionamento é usado um ponto no final que representa todos os arquivos.

git commit -a -m "Description do commit"

Cria um novo commit com todas as alterações adicionadas no fluxo de versionamento (git add .).
O parâmetro -a serve para instruir o Git adicionar o commit para todas as alterações do fluxo.
O parâmetro -m é usado para adicionar uma mensagem de commit, que seria a descrição do commit que está tentando enviar.

git remote add origin https://github.com/lucasWeirich/learning_git.git

Adiciona um repositório remoto ao repositório Git local.
Nome "origin" é usado para se referir a esse repositório remotamente adicionado.
Você está dizendo ao Git para adicionar um novo repositório remoto com o nome "origin" e o URL fornecido.

git push -u origin main

Envia (push) as alterações do branch atual do repositório Git local para a branch principal do repositório remoto "origin".
O parâmetro "-u" significa "upstream" e é usado para configurar a branch local para rastrear a branch remota. Isso significa que, após executar esse comando, sempre que você executar o comando "git push" sem especificar o branch, o Git saberá que você deseja enviar as alterações do branch local para a branch remota configurada com o upstream.
No caso específico do comando "git push -u origin main", você está enviando as alterações do branch local atual para a branch "main" do repositório remoto "origin" e configurando a branch local para rastrear a branch remota "main" como upstream.

git branch -M main

Renomeia a branch padrão do repositório Git de "master" para "main".
Git irá renomear a branch atual do repositório para "main". Isso significa que, a partir desse momento, a branch padrão do seu repositório será "main" e não "master".

git checkout -b "minhaBranch"

Utilizado para mudar para a branch (ramificação) em um repositório Git.
Quando utilizado -b ele cria uma nova branch com o nome informado.
